### 1. 비주얼 시스템 확정 (Visual System Finalization)

**A. 컬러 팔레트 (Color Palette)**
| 역할 | 색상명 | HEX 코드 | 용도 |
| :--- | :--- | :--- | :--- |
| **Primary (Control/Authority)** | Deep Navy | `#0A1931` | 배경, 주요 제목, 시스템 프레임 (권위 및 안정성) |
| **Accent (System/Action)** | Bright Cyan | `#00FFFF` | 핵심 CTA 버튼, 데이터 흐름 강조, 통제 지표 (에너지 및 기술) |
| **Secondary (Text/Contrast)** | Light Gray | `#CCD6E1` | 본문 텍스트, 보조 정보 (가독성 확보) |
| **Background** | Dark Slate | `#1A2330` | 전체 페이지 배경 (Navy와 대비를 주어 깊이감 부여) |

**B. 타이포그래피 (Typography)**
*   **Header/Title:** Inter (Bold, Large Size) - 시스템의 강력함 강조
*   **Body Text:** Roboto or Open Sans (Regular/Medium) - 높은 가독성 확보
*   **Accent Data:** Monospace Font (예: Space Mono) - 데이터 및 코드의 정밀함 표현

**C. 핵심 레이아웃 컨셉 (Layout Concept: The Control Panel)**
페이지 전체는 **'시스템 인터페이스(UI)'** 느낌을 극대화하여 디자인합니다.

1.  **Navigation Bar:** 상단에 네이비 배경으로 시스템 메뉴와 현재 상태 표시기(Status Indicator)를 배치합니다.
2.  **Central Focus (The Control Panel):** 페이지 중앙에 가장 크고 시각적으로 두드러지게 'Control Panel'의 메인 대시보드를 배치합니다. 이는 데이터 흐름과 통제권을 시각화하는 핵심 요소입니다.
3.  **Information Flow:** 좌측 또는 우측에 시스템 구성 요소(Input, Process, Output)를 네이비 라인으로 연결하고, 각 단계별로 Cyan 색상의 지표(Gauge, Bar, Status Light)를 사용하여 프로세스의 투명성과 안정성을 표현합니다.

### 2. Silver Tier 랜딩 페이지 레이아웃 상세 스펙 (Figma Implementation Guide)

*   **메인 비주얼:** 'System Blueprint'의 핵심 기하학적 구조(네이비/사이안 라인)를 배경으로 미묘하게 오버레이. 중앙에 큰 타이틀 배치.

*   **Sub-Headline:** "시스템 통합 거버넌스가 불확실성을 제거하고, 무한한 확장성을 보장합니다."

**[Section 2: Pain Point & Solution - 문제점과 통제권의 전환]**

*   **Pain Points (좌측):** 리스크, 불확실성, 비효율성 등의 키워드를 작은 아이콘과 함께 나열합니다.
*   **Solution Visual (우측):** '통제(Control)', '투명성(Transparency)', '안정성(Stability)' 세 가지 핵심 가치를 Cyan 색상의 대시보드 형태로 배치하여 시각적 해결책을 제시합니다.

**[Section 3: Feature Deep Dive - 거버넌스 구조]**
*   **컨셉:** 'System Blueprint' 목업(`sessions/2026-05-04T09-14/designer.md`)을 기반으로 각 기능 섹션을 구성합니다.
*   **디자인 요소:** 각 기능(예: Audit Log, Risk Assessment)은 네이비 박스 안에 Cyan 라인으로 연결된 인터페이스 형태로 표현하여, 단순한 리스트가 아닌 '시스템의 일부'처럼 보이게 합니다.
